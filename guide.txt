WEZTERM SESSION MODEL (unified — everything is persistent)
==========================================================

There is only ONE concept now: the workspace. The GUI auto-connects to the
background mux server on launch (config.default_gui_startup_args), and that
server auto-starts itself the first time — so there is NO manual
`wezterm-mux-server --daemonize` step any more.

Because every tab runs on the mux server, every workspace/session survives a
full WezTerm close. "Session" and "workspace" are the same thing.

Daily use:
  Leader + a   create or switch to a named session (blank name = "default")
  Leader + w   toggle back to your previous session (bounce between two)
  Leader + '   session hub: pick a session to switch instantly; create and
               manage (rename / delete) live behind their own entries
  Leader + d   close the current tab (its session stays on the server)
  Leader + g   delete a tracked session

To step away and keep everything running:
  Just close the WezTerm window (or Leader+Shift+q to quit the GUI).
  The mux server keeps all panes alive in the background. Relaunch WezTerm
  and it reattaches to exactly what you left — no window duplication.

Notes:
  - Leader+a no longer spawns a second WezTerm window. That was AttachDomain
    re-importing every existing mux window; it has been removed.
  - The mux server only dies if you kill wezterm-mux-server.exe or reboot.
